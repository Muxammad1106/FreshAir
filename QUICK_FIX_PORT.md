# Быстрое решение: порт уже занят

## Проблема
Порты 8000 (backend) и 3000 (frontend) уже заняты другими процессами.

## Решение
Порты по умолчанию изменены:
- **Backend**: 8000 → **8001**
- **Frontend**: 3000 → **8080**

## Решение 1: Изменить порт в .env

Отредактируйте файл `.env`:

```bash
nano .env
```

Измените `FRONTEND_PORT` на свободный порт:
```env
FRONTEND_PORT=8080
```

Или другой порт, например:
- `8080`
- `8081`
- `3001`
- `80` (требует sudo, используется для production)

## Решение 2: Остановить процесс на порту 3000

Найдите процесс:
```bash
sudo lsof -i :3000
# или
sudo netstat -tulpn | grep :3000
```

Остановите процесс:
```bash
# Если это старый контейнер
docker ps | grep 3000
docker stop <container_id>

# Или убейте процесс
sudo kill -9 <PID>
```

## Решение 3: Использовать другой порт в docker-compose

Временно измените в `docker-compose.yml`:
```yaml
ports:
  - "8080:80"  # Вместо 3000:80
```

## Проверка свободных портов

```bash
# Проверить занятые порты
sudo netstat -tulpn | grep LISTEN

# Или
sudo ss -tulpn | grep LISTEN
```

## Production

Для production рекомендуется использовать Nginx reverse proxy на порту 80/443, а не прямой доступ к контейнеру.
